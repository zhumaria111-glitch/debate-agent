**Query**：罗德柴尔斯 青霉素

**预期命中**：技术是否中立（反方原文引用罗氏死于细菌感染的事例）

### 结果

**检索失败。** 检索到的 top 5 段落中虽然包含"技术是不是中立的？"，但未命中包含"罗斯柴尔德"和"青霉素"的段落。AI 无法找到目标内容，报告知识库中没有相关记录。

## 发现与建议

### 关键发现

1. **检索偏斜问题严重**：查询 6 和 8 的 top 5 全部来自同一场辩论，未能跨视频召回。当前的 `search()` 方法对每层（transcripts/claims/debates）独立取 top n，但没有保证结果覆盖多场辩论的机制。

2. **译名/隐喻盲区**：这是语义检索的固有问题。384 维 MiniLM 对"罗德柴尔斯→罗斯柴尔德"和诗歌化比喻的跨表述匹配不足。解决方案：
   - 添加同义词/译名词典做 query expansion
   - 升级到更大维度的 embedding 模型（如 768 维）
   - 对查询做拼音/字形层面的模糊匹配补充

3. **`_chunk_text` 死循环 bug**（已修复）：当段落中第一个"。"出现在前 100 字符内时，overlap 计算导致 `start=0`，`current = current[0:]` 不变，形成无限循环。修复为 `if start <= 0: start = split_at`。

4. **AI 回答质量在检索命中时很高**：当 KB 返回了正确段落时，AI 的引用、分析和延伸都表现出色。系统的瓶颈在检索端而非生成端。

### 优先级建议

| 优先级 | 改进项 | 预期提升 |
|--------|--------|----------|
| P0 | 跨视频结果多样性：search() 中按 debate_id 分散采样 | 跨视频对比 2.5 → 3.5 |
| P1 | Query expansion：译名词典 + 同义词表 | 命中率 3.5 → 4.0 |
| P1 | 升级 embedding 模型：multilingual-e5-large（1024 维） | 细节检索 3.0 → 3.5 |
| P2 | 混合检索权重调优：提高 keyword boost 系数 | 命中率小幅提升 |
| P2 | 在 chunk 内容前添加更丰富的上下文标签 | 帮助 embedding 定位 |

### 与 v0.3.1/v0.4.2 的联动

- v0.3.1（结构化质量）：综合分 4.24/5
- v0.4.2（paste-URL pipeline + cleaning）：综合分 3.8/5
- **v0.5（RAG 知识库）**：综合分 **3.4/5**

趋势：结构化分析（单视频）表现最好，pipeline 自动化引入了一些衰减，RAG 知识库作为初版功能达到了可用水平，但跨视频对比和细节检索有明确的改进空间。
