| 改动 | 类型 | 针对问题 |

| merge prompt 加硬约束：top3 必须恰好 3 条、clash/unresolved 禁止为空 | prompt | quick_view 三四五系统性为空 |

## 迭代叙事（面试素材）

> **v0.2 → v0.3 → v0.3.1 三次迭代，综合分从 2.72 提升到 4.12。**
>
> **第一轮（v0.3）解决数据完整性问题**：诊断出 `\n\n` 碎片化 + max_tokens 截断导致辩论三四覆盖率不到 10%，通过预处理分段 + max_tokens 翻倍 + 解析报错重试，覆盖率恢复到 100%，综合分提至 3.52。
>
> **第二轮（v0.3.1）解决摘要质量问题**：发现 quick_view 的 Top3/交锋/未回应在 chunked 路径下系统性为空——根因是 merge prompt 缺乏硬约束，LLM 随机选择是否填充。在 prompt 中加"必须恰好 3 条""禁止返回空数组/空字符串"约束，同时加规则过滤评委发言。两行 prompt 改动，辩论三四从 3.4/3.2 升至 4.4，达到与 single-pass 基线持平的水平。
>
> **关键 takeaway**：最难的 bug（回合丢失）需要代码层修复，但影响最大的修复（quick_view 空白）只需要两行 prompt 改动。AI 产品的迭代不是越改越多代码，而是越改越准的 prompt。
