# 产品需求文档（PRD）

## 1. 产品愿景

让每一场辩论赛都能被看懂。粘贴一个视频链接，三分钟看清论证骨架，追问任何你好奇的问题。

---

## 2. 目标用户与场景

**首发用户**：陈琳式的入门辩手——靠看比赛自学，投入时间多但收获薄。

**核心场景**：晚上在宿舍打开一场辩论赛视频，看完不知道谁说了什么。把链接丢进产品，三分钟看清正反方各站什么立场、最关键的攻防在哪里。然后挑自己没想通的地方追问 AI——"反方那个数据站得住吗？""正方为什么不回应 XX？"

详见 [用户画像文档](./user-personas.md)。

---

## 3. 核心功能

### P0 — 没有这些产品不成立

| 功能 | 描述 | 用户价值 |
|------|------|---------|
| **视频链接输入** | 用户粘贴 B 站 / YouTube 链接，自动获取字幕/转写为文字稿 | 消灭使用门槛。用户不需要自己找文字稿 |
| **3 分钟速览** | 辩题、双方立场、Top 3 论点、最激烈交锋、关键未回应问题 | 核心价值。一个小时内化成三分钟 |
| **AI 深度追问** | 多轮对话，支持搜索原文、追踪攻防链、对比立场、核查事实、辩手分析 | 从"看懂"到"学会"。用户参与感 |

### P1 — 有了更好

| 功能 | 描述 | 用户价值 |
|------|------|---------|
| **多类型 sample 辩论** | 内置政策辩/价值辩/事实辩各一篇 sample，新用户零门槛体验 | 新用户不需要自己找链接就能体验产品 |
| **分析报告导出** | 一键导出 Markdown 格式分析报告 | 留存。用户可以保存、分享、积攒学习档案 |
| **使用引导** | 侧边栏简要说明产品怎么用、什么样的视频效果好 | 降低新用户困惑 |

### P2 — 下个版本

| 功能 | 描述 |
|------|------|
| 音视频直接上传 | 支持 mp3/wav/mp4 文件上传，自动转文字 |
| 历史分析记录 | 保存用户的历次分析，可回顾 |
| 论点可视化图 | 正反方论点 + 反驳关系的可视化展示 |
| 任意 AI 对话导出 | 不止报告，整段对话可导出 |

## 4. 用户流程

```
用户打开产品
    │
    ├─ 新用户 ──→ 点击"加载示例"体验完整流程
    │
    └─ 任何用户
          │
          ▼
    粘贴 B站/YouTube 视频链接（或粘贴文字稿）
          │
          ▼
    点击"开始分析"（等待 15-30 秒）
          │
          ▼
    看到"3分钟速览卡片"
    ├─ 辩题 + 双方立场
    ├─ 正方 Top 3 论点  |  反方 Top 3 论点
    ├─ 最激烈交锋点
    ├─ 关键未回应问题
    └─ 关键词
          │
          ▼
    看到 AI 欢迎语 + 建议追问方向
          │
          ▼
    用户输入问题（或点击建议问题）
          │
          ▼
    AI 搜索辩论数据，返回分析
          │
          ▼
    用户可以继续追问（无限轮）
          │
          ▼
    完成 → 导出报告 / 分析新的辩论
```

## 5. 成功指标

| 指标 | 定义 | 为什么重要 |
|------|------|-----------|
| **首次体验完成率** | 粘贴链接 → 看到速览卡片 → 至少追问一个问题 | 证明用户走完了核心循环，体验到了价值 |
| **追问深度** | 每个会话的平均追问轮数 | 追问越多，说明速览激发了用户的思考，参与感在起作用 |
| **口口相传意愿** | "你会把这个工具推荐给队友吗？"（用户测试后直接问） | 最朴素的产品验证。用户愿意推荐=真的有价值 |

**不做**的指标：注册量、DAU、留存率——当前阶段不需要，这是单人学习工具不是社交产品。

---

## 6. 不做的事

| 不做 | 理由 |
|------|------|
| **AI 实时陪练（跟 AI 打辩论）** | 竞品扎堆的方向。我的差异化恰恰是"分析"不是"对战"。做陪练会模糊定位 |
| **社区功能（评论/点赞/分享广场）** | 当前阶段不需要网络效应。先让一个人能用爽，再考虑多人 |
| **音频/视频直接上传** | 输入问题用"视频链接"已经解决。直接上传涉及 ASR 集成+说话人分离，工程量大但 PM 价值低。P2 再做 |
| **多语言支持** | 首发只做中文辩论。英文辩论市场有 Stela AI 等产品，先不碰 |
| **移动端 App** | Streamlit 本身响应式，手机浏览器够用。原生 App 的 ROI 在早期极低 |
| **付费墙** | 找实习用的作品，有用户反馈比有钱重要。免费开放 |

## 7. 关键产品决策记录

**决策 1：输入方式选视频链接而不是文字稿粘贴。**
- 原因：目标用户看的是 B 站视频，不是抱着文字稿看。要求用户先找稿子再粘贴，多了一步就多一层流失。
- 风险：视频链接 → 字幕获取这一步依赖上游平台，技术上可能不稳定。
- 应对：文字稿粘贴作为保底入口保留。

**决策 2：首发用户选入门辩手而不是辩论教练或研究者。**
- 原因：入门辩手最多、最容易触达、使用频率最高、对产品最宽容。
- 详见用户画像文档。

**决策 3：产品形态是 Web App 而不是浏览器插件或小程序。**
- 原因：Web App 开发最快（Streamlit 已有 demo），适合快速验证。浏览器插件需要额外开发成本。小程序需要审核。
